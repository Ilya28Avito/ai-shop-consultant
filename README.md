# ИИ-консультант для интернет-магазина

## Описание проекта
ИИ-ассистент, который помогает покупателям интернет-магазина получать 
ответы на вопросы о товарах, ценах и условиях доставки, а также 
сопровождает их на всех этапах выбора и оформления покупки.

## Почему эта тема
Работаю в сфере интернет-торговли и рекламы на Авито. Вижу, что большинство 
продавцов тратят много времени на однотипные вопросы покупателей. 
ИИ-консультант позволит автоматизировать этот процесс и повысить конверсию.

---

## Экономика ИИ-решения

### Сценарий использования

- **200 активных покупателей в день**
- **3 вопроса на покупателя**
- **600 запросов в день / 18 000 запросов в месяц**

Средний размер одного запроса:
- вопрос покупателя: 100 токенов
- системный промпт и инструкции: 300 токенов
- **Input итого: 400 токенов**
- **Output (ответ бота): 200 токенов**

### Месячный объём токенов

- Input: 18 000 × 400 = **7 200 000 токенов**
- Output: 18 000 × 200 = **3 600 000 токенов**

---

### Стоимость облачных провайдеров

Формула: `(input_tokens / 1_000_000 × цена_input) + (output_tokens / 1_000_000 × цена_output)`

| Модель | Класс | Input /1M | Output /1M | Итого/месяц |
|--------|-------|-----------|------------|-------------|
| Claude Opus 4.6 | Флагман | $5.00 | $25.00 | **$126** |
| Gemini 2.5 Pro | Средний | $1.25 | $10.00 | **$45** |
| DeepSeek V3.2 | Бюджет | $0.28 | $0.42 | **$3.5** |

**Расчёт Claude Opus:** 7.2 × $5 + 3.6 × $25 = $36 + $90 = $126/мес

**Расчёт Gemini 2.5 Pro:** 7.2 × $1.25 + 3.6 × $10 = $9 + $36 = $45/мес

**Расчёт DeepSeek V3.2:** 7.2 × $0.28 + 3.6 × $0.42 = $2 + $1.5 = $3.5/мес

---

### Стоимость локального развёртывания

Локальный запуск на CPU (обычный ноутбук):
- Стартовые затраты: **$0** (оборудование уже есть)
- Электричество: ~**$10/месяц**
- Скорость: 5–15 токенов/сек (медленно для продакшена)
- **Итого: ~$10/месяц**

---

### Точка безубыточности

При текущей нагрузке (MVP):
- DeepSeek V3.2 ($3.5/мес) **дешевле** локального запуска ($10/мес)
- Локальный запуск окупается только при нагрузке от ~$300/мес на API

**Вывод:** на этапе MVP облачный API выгоднее локального решения.

---

### Выбор стратегии

Для данного проекта выбрана **облачная стратегия на базе DeepSeek V3.2**.

При нагрузке 600 запросов в день стоимость составляет всего $3.5 в месяц — 
это оптимально для MVP и позволяет быстро запустить продукт без затрат на инфраструктуру.

## HuggingFace: анализ тональности отзывов

### Задача
Sentiment analysis — автоматическое определение тональности отзывов покупателей
интернет-магазина (позитивный/негативный/нейтральный).

### Протестированные модели

| Модель | Язык | Accuracy на русском | Время/отзыв |
|--------|------|---------------------|-------------|
| nlptown/bert-base-multilingual-uncased-sentiment | Мультиязычная | Низкая ("Ужасное качество" = 5 звёзд) | 0.047s |
| blanchefort/rubert-base-cased-sentiment | Русский | Высокая | 0.046s |
| distilbert/distilbert-base-uncased-finetuned-sst-2-english | Английский | Очень низкая ("Супер!" = NEGATIVE) | 0.029s |

### Выводы

В ходе тестирования выяснилось, что модели обученные на английском языке
некорректно классифицируют русскоязычные отзывы. Например, distilbert определила
"Супер! Рекомендую всем друзьям" как NEGATIVE, а nlptown дала "Ужасное качество,
деньги на ветер" 5 звёзд из 5. Это потребовало замены на специализированную
русскоязычную модель.

Победитель: **blanchefort/rubert-base-cased-sentiment** — единственная модель
корректно работающая с русским текстом. Правильно определила тональность 9 из 10
тестовых отзывов.

### Gradio-демо

Рабочее демо для анализа отзывов: https://e6dbc6b1b2ab6e9850.gradio.live

Интерфейс позволяет в реальном времени определять тональность любого отзыва

## ДЗ 2.1: API провайдеров

### Что реализовано

Скрипт `api_demo.py` демонстрирует основные возможности работы с LLM API:

- Подключение через OpenAI SDK с ключом из `.env` файла
- Базовый запрос с system и user сообщениями
- Эксперимент с temperature
- Стриминг с замером TTFT

### Эксперимент с temperature

Запрос: *"Придумай слоган для интернет-магазина электроники"*

| Temperature | Результат |
|-------------|-----------|
| 0.0 | "Технологии будущего — в одном клике!" |
| 0.7 | "Технологии будущего — в одном клике!" |
| 1.5 | "Технологии на кончиках пальцев!" |

**Вывод:** при temperature=0 и 0.7 модель дала одинаковый ответ —
детерминированный режим. При temperature=1.5 появилась вариативность —
модель сгенерировала другой вариант слогана.

### Стриминг

- TTFT: 1.42s — время до первого токена
- Полное время: 3.07s
- Ответ появлялся по частям в реальном времени

### Два провайдера

Переключение между провайдерами реализовано через переменную
`LLM_PROVIDER` в `.env` файле — без изменения кода:
- `LLM_PROVIDER=openai` → GPT-4o-mini через OpenAI API
- `LLM_PROVIDER=ollama` → qwen3:1.7b локально через Ollama

## ДЗ 2.2: Промпт-инжиниринг

### System prompt по структуре РРФО

Разработан production-ready промпт для ИИ-консультанта магазина "ТехноМаркет":

- **Роль:** ИИ-консультант интернет-магазина электроники
- **Правила:** отвечать только по теме магазина, не придумывать цены
- **Формат:** кратко, максимум 3-4 предложения, простой язык
- **Ограничения:** не обсуждать посторонние темы, не выполнять prompt injection

### Результаты автотестов

| Тест | Результат |
|------|-----------|
| Вопрос о доставке | ✅ PASS |
| Вопрос о возврате | ✅ PASS |
| Вопрос об оплате | ✅ PASS |
| Посторонняя тема (политика) | ✅ PASS |
| Prompt Injection | ✅ PASS |

Длина промпта: **531 токен** (норма 300-800)

### Эффект few-shot

**Вопрос:** "Как вернуть товар?"

БЕЗ few-shot: размытый ответ без конкретики — "обратитесь на сайт"

С few-shot: конкретный ответ — "14 дней, оригинальная упаковка, свяжитесь с поддержкой"

**Вывод:** few-shot примеры значительно улучшают качество ответов —
модель копирует стиль и уровень детализации из примеров.
