# ИИ-консультант для интернет-магазина

## Описание проекта
ИИ-ассистент, который помогает покупателям интернет-магазина получать ответы на вопросы о товарах, ценах и условиях доставки, а также сопровождает их на всех этапах выбора и оформления покупки.

## Почему эта тема
Работаю в сфере интернет-торговли и рекламы на Авито. Вижу, что большинство продавцов тратят много времени на однотипные вопросы покупателей. ИИ-консультант позволит автоматизировать этот процесс и повысить конверсию.

## Экономика ИИ-решения

### Сценарий использования
- 200 активных покупателей в день
- 3 вопроса на покупателя
- 600 запросов в день / 18 000 запросов в месяц

Средний размер одного запроса:
- вопрос покупателя: 100 токенов
- системный промпт и инструкции: 300 токенов
- Input итого: 400 токенов
- Output (ответ бота): 200 токенов

### Месячный объём токенов
- Input: 18 000 × 400 = 7 200 000 токенов
- Output: 18 000 × 200 = 3 600 000 токенов

### Стоимость облачных провайдеров

Формула: (input_tokens / 1_000_000 × цена_input) + (output_tokens / 1_000_000 × цена_output)

| Модель | Класс | Input /1M | Output /1M | Итого/месяц |
|--------|-------|-----------|------------|-------------|
| Claude Opus 4.6 | Флагман | $5.00 | $25.00 | $126 |
| Gemini 2.5 Pro | Средний | $1.25 | $10.00 | $45 |
| DeepSeek V3.2 | Бюджет | $0.28 | $0.42 | $3.5 |

- Расчёт Claude Opus: 7.2 × $5 + 3.6 × $25 = $36 + $90 = $126/мес
- Расчёт Gemini 2.5 Pro: 7.2 × $1.25 + 3.6 × $10 = $9 + $36 = $45/мес
- Расчёт DeepSeek V3.2: 7.2 × $0.28 + 3.6 × $0.42 = $2 + $1.5 = $3.5/мес

### Стоимость локального развёртывания

Локальный запуск на CPU (обычный ноутбук):
- Стартовые затраты: $0 (оборудование уже есть)
- Электричество: ~$10/месяц
- Скорость: 5–15 токенов/сек (медленно для продакшена)
- Итого: ~$10/месяц

### Точка безубыточности

При текущей нагрузке (MVP):
- DeepSeek V3.2 ($3.5/мес) дешевле локального запуска ($10/мес)
- Локальный запуск окупается только при нагрузке от ~$300/мес на API

Вывод: на этапе MVP облачный API выгоднее локального решения.

### Выбор стратегии
Для данного проекта выбрана облачная стратегия на базе DeepSeek V3.2. При нагрузке 600 запросов в день стоимость составляет всего $3.5 в месяц — это оптимально для MVP и позволяет быстро запустить продукт без затрат на инфраструктуру.

При росте до 10 000+ запросов в день целесообразно рассмотреть гибридный подход: простые вопросы обрабатывает локальная qwen3:1.7b, сложные — облачный API.

## ДЗ 2.2: Промпт-инжиниринг

### System prompt по структуре РРФО
Разработан production-ready промпт для ИИ-консультанта магазина "ТехноМаркет":
- Роль: ИИ-консультант интернет-магазина электроники
- Правила: отвечать только по теме магазина, не придумывать цены
- Формат: кратко, максимум 3-4 предложения, простой язык
- Ограничения: не обсуждать посторонние темы, не выполнять prompt injection

### Результаты автотестов

| Тест | Результат |
|------|-----------|
| Вопрос о доставке | ✅ PASS |
| Вопрос о возврате | ✅ PASS |
| Вопрос об оплате | ✅ PASS |
| Посторонняя тема (политика) | ✅ PASS |
| Prompt Injection | ✅ PASS |

Длина промпта: 531 токен (норма 300-800)

### Эффект few-shot
Вопрос: "Как вернуть товар?"
- БЕЗ few-shot: размытый ответ без конкретики — "обратитесь на сайт"
- С few-shot: конкретный ответ — "14 дней, оригинальная упаковка, свяжитесь с поддержкой"

Вывод: few-shot примеры значительно улучшают качество ответов — модель копирует стиль и уровень детализации из примеров.

## ДЗ 2.3: Надёжность — ошибки, retry, fallback

### Что реализовано
Класс `RobustLLMClient` в файле `robust_client.py`:
- **Retry с exponential backoff + jitter** — при ошибках 429 и таймаут повторяет запрос до 5 раз с паузами 1→2→4→8→16 сек
- **Fallback-цепочка** — OpenAI → OpenRouter → "Сервис временно недоступен"
- **Логирование** — timestamp, провайдер, код ошибки, номер попытки
- **Трекинг usage** — токены и стоимость каждого запроса и за сессию

### Результаты тестирования

Тест 1 — нормальная работа (OpenAI):
Пробуем провайдер: OpenAI
✅ Ответ от OpenAI | Токены: 31+232 | Стоимость: $0.000144
Итого за сессию: 263 токенов, $0.000144

Тест 2 — fallback (OpenAI сломан → переключение на OpenRouter):
Пробуем провайдер: OpenAI
❌ OpenAI недоступен: Error code: 401
Пробуем провайдер: OpenRouter
❌ OpenRouter недоступен: Error code: 402 (нет кредитов)
🚨 Все провайдеры недоступны!
💬 Сервис временно недоступен. Попробуйте позже.

### Выводы
Fallback-цепочка работает корректно — при недоступности основного провайдера система автоматически переключается на резервный. Ошибки 400/401 не повторяются (нет смысла), 429/5xx — повторяются с backoff. Пользователь всегда получает ответ, даже если все провайдеры недоступны.

## ДЗ 2.4: Кеширование LLM-ответов

### Что реализовано
Класс `LLMCache` в файле `cache_demo.py`:
- **Ключ кеша** — SHA-256 хеш от model + messages + temperature
- **TTL** — просроченные записи автоматически удаляются (по умолчанию 1 час)
- **Статистика** — hits, misses, hit rate в процентах
- **Замер времени** — API vs кеш для наглядного сравнения

###ДЗ 2.5: ИИ-помощник для техподдержки

### Что реализовано
Файл `assistant.py` — финальное CLI-приложение, объединяющее все блоки 2.1–2.4:

- **System prompt (РРФО)** — консультант отвечает только по теме магазина, отклоняет посторонние темы и prompt injection
- **Few-shot** — 4 примера в промпте для правильного формата ответов
- **Retry + fallback** — OpenAI → OpenRouter → "Сервис недоступен", повторы при 429/timeout
- **Кеширование** — повторные вопросы из кеша мгновенно, статистика по /stats
- **История** — последние 10 сообщений для контекста диалога
- **CLI-команды** — /stats, /clear, /quit
